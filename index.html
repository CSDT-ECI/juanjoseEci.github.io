Proyecto del Curso

Ferry legacy es una apliación simple propuesta a manera de reto alojada en el siguiente repositorio Github.
Ferry legacy es un proyecto para refinar nuestras habilidades en la detección de smell codes con una deuda técnica grave. Este proyecto posee varios aspectos a revisar como documentación , segmentación , pruebas unitarias entre otras que iremos revisando a lo largo del curso.

  • - Java & C#
  • - Gradle
  • - SonarLint
  • - Github Copilot
  • - Designite-Java

Arquitectura

Carece de un estilo arquitectonico adecuado para el proyecto, lo cual es crucial para la mantenibilidad del proyecto.

Deuda tecnica

Este código legacy en todas sus clases posee Smell Codes lo que indican malas practicas como por ejemplo la falta de documentación , alto acoplamiento, errores de sintaxis entre otros.

Atributos de Calidad

Los atributos de calidad violados que más se repiten en los analisis del proyecto son : Mantenibilidad , Eficiencia, Usabilidad y Fiabilidad

Calidad del Software

La aplicación aunque funcione no representa un producto de calidad ya que posee una deuda tecnica grave, fue analizado de manera rigurosa por diferentes aplicaciones.

Ver Tecnologías

SonarLint es una herramienta de analisis de código para la deteccion de smell codes en Java y otros lenguajes, pieza clave en este proyecto para corregir y presentar un producto de calidad.

GitHubCopilot es una inteligencia artificial que ayuda a mejorar la eficiencia a la hora de programar, funcinonando como un asistente el cual tiene la capacidad de realizar tareas como autogenerar código ahorrando tiempo, en este caso a la hora de corregir issues repetitivos o autogenerar los tests.

Designite es una herramienta de análisis de código y calidad de software que permite identificar y resolver problemas de calidad en el proyecto. Proporciona una serie de características útiles, incluyendo la detección de "olores de código", "olores de arquitectura" y "olores de implementación", que son patrones problemáticos en el diseño y la implementación del software.

Intellij es una interprete de lenguajes de programación el cual fue usado en este proyecto para editar y visualizar los componentes del mismo , adionalmente fue esencial debido a que la herramienta SonarLint a manera de plugin solo es compatible con este IDE permitiendo de una forma sencilla la refactorización del código.
